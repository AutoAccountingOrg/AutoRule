# 编写说明

本目录的规则格式为 `支付宝xxx`表示具体的支付宝支付服务，如果该账单来自于支付宝账单列表，请合并进`支付宝账单列表`.
例如：
- `支付宝亲情卡`

# 渠道（Channel）命名格式

`支付宝[服务名称-收支类型]`，例如：

- `支付宝[亲情卡-支出]`
- `支付宝[亲情卡-收入]`

# 交易账户命名格式

一般交易账户不需要修改命名，但是一些特殊的比较容易有歧义的需要重新命名。

例如：

- `余额` -> `支付宝余额`

# 测试用例的选用

所有的测试用例应该尽量避免重复，减少维护工作。

例如：有多个用户提交了同一种格式的账单，只需要选取其中一个作为测试用例即可。
